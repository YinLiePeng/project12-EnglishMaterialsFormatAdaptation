## 🧪 手动测试步骤

### 1. 启动后端服务

```bash
cd backend
source venv/bin/activate  # Windows: venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端将在 http://localhost:8000 运行

### 2. 启动前端服务

```bash
cd frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

前端将在 http://localhost:5173 运行

### 3. 测试场景1：快速修正流程

**操作步骤：**

1. 访问 http://localhost:5173
2. 上传一个测试文档（使用 `测试用例/教师从网上下载的资料样例/` 下的文档）
3. 等待文档处理完成
4. 点击"查看详情"进入DocumentPreview页面
5. 在右侧结构识别结果列表中：
   - 点击任意段落的类型标签（例如"正文"）
   - 从下拉菜单中选择新类型（例如"题号"）
   - 观察是否出现 ✏️ 编辑标记
   - 观察顶部是否显示"已编辑 N 处"
6. 点击"💾 保存并重新生成"按钮
7. 等待处理完成
8. 下载新生成的文档
9. 验证修改是否生效

**预期结果：**
- ✅ 段落类型可以成功修改
- ✅ 编辑标记显示正确
- ✅ 保存后自动重新生成文档
- ✅ 新文档的样式符合预期

### 4. 测试场景2：AI识别流程

**操作步骤：**

1. 在DocumentPreview页面
2. 点击"😕 结果不满意？"展开折叠面板
3. 在"📝 说明问题（可选）"输入框中填写意见，例如：
   - "段落0应该是标题"
   - "所有以1.开头的都是题号"
4. 点击"🤖 AI识别"按钮
5. 等待2-10秒（AI处理时间）
6. 查看"确认AI识别修正"对话框：
   - 检查变化列表是否正确
   - 检查变化理由是否合理
7. 点击"确认并重新生成文档"按钮
8. 等待处理完成
9. 下载新生成的文档
10. 验证修改是否生效

**预期结果：**
- ✅ AI识别能够基于用户反馈优化结果
- ✅ 预览对话框显示所有变化
- ✅ 确认后自动重新生成文档
- ✅ 新文档符合AI识别的结果

### 5. 测试场景3：组合操作

**操作步骤：**

1. 先使用快速修正修改几个段落
2. 保存并重新生成
3. 再使用AI识别进行优化
4. 再次使用快速修正调整个别段落
5. 最终下载文档验证

**预期结果：**
- ✅ 快速修正和AI识别可以交替使用

## 🔍 故障排查

### 问题1：后端启动失败

**可能原因：**
- Python虚拟环境未激活
- 依赖包未安装
- 端口8000被占用

**解决方案：**
```bash
cd backend
python -m venv venv  # 如果虚拟环境不存在
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 问题2：前端启动失败

**可能原因：**
- node_modules未安装
- 端口5173被占用

**解决方案：**
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 问题3：API调用失败

**检查项：**
- 后端服务是否正常运行（访问 http://localhost:8000/docs）
- 浏览器控制台是否有错误信息
- 网络请求是否返回错误

### 问题4：文档重新生成失败

**检查项：**
- 查看后端日志
- 检查 `backend/tmp/output/` 目录权限
- 确认原始文档文件存在
