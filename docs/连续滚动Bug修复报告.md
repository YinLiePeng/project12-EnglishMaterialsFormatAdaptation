# 连续滚动Bug修复 - 完成报告

## 🐛 问题描述

### 现象
用户报告："光标放在处理后文档的某些地方的时候，文档就会连续的往上滚动到顶或者往下滚动到底。"

### 影响
- 用户体验极差
- 无法正常浏览文档
- 严重影响功能使用

---

## 🔍 根本原因分析

### 事件循环链路

```
用户鼠标悬停在右侧段落
    ↓
触发 handleRightParagraphHover
    ↓
调用 scrollIntoView() 滚动左侧文档
    ↓
触发左侧 scroll 事件
    ↓
触发 handleLeftScroll (同步滚动)
    ↓
滚动右侧文档
    ↓
右侧文档移动，鼠标悬停到新段落
    ↓
再次触发 handleRightParagraphHover
    ↓
无限循环 → 连续滚动
```

### 问题代码（修复前）

**位置**：`frontend/src/pages/DocumentPreview.tsx` 第148-154行

```typescript
const handleRightParagraphHover = useCallback((index: number) => {
  setHoveredParagraph(index);
  const leftItem = document.getElementById(`left-item-${index}`);
  if (leftItem && leftPanelRef.current) {
    leftItem.scrollIntoView({ behavior: 'smooth', block: 'center' }); // ❌ 导致循环
  }
}, []);
```

**问题**：
- `scrollIntoView()` 滚动左侧文档
- 触发同步滚动机制
- 同步滚动导致右侧文档移动
- 鼠标悬停到新段落
- 再次触发 `scrollIntoView()`
- 形成死循环

---

## ✅ 修复方案

### 方案选择

采用**方案1：完全移除自动滚动**

### 修复代码

**修改后**：

```typescript
const handleRightParagraphHover = useCallback((index: number) => {
  setHoveredParagraph(index);
  // 移除自动滚动功能，避免事件循环导致连续滚动
  // 只保留悬停高亮功能，配合同步滚动使用
}, []);
```

**改进**：
- ✅ 移除了 `scrollIntoView()` 调用
- ✅ 保留了 `setHoveredParagraph(index)` 高亮功能
- ✅ 添加了清晰的注释说明

## 💡 设计理念

### 为什么移除自动定位？

1. **同步滚动已足够**
   - 用户手动滚动时，两侧会自动同步
   - 已经提供了很好的对比体验
   - 自动定位反而是多余的

2. **避免意外滚动**
   - 用户只是想查看某个段落
   - 不希望文档突然滚动
   - 移除自动定位后更符合预期

3. **简化交互**
   - 悬停 = 查看信息（高亮+tooltip）
   - 滚动 = 手动控制
   - 职责清晰，不会冲突
