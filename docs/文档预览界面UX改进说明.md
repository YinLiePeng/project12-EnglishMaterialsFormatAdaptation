# 文档预览界面UX改进说明

## 📋 改进概述

将文档预览界面从"文档+结构信息"模式改为"处理前+处理后"对比模式，提升用户体验，让用户更直观地看到格式适配的效果。

---

## 🎯 核心改进

### 1. **右侧改为处理后文档预览**

**之前**：
- 右侧显示结构信息列表
- 需要点击"详情"才能看到样式
- 用户无法直观看到处理后的效果

**改进后**：
- 右侧渲染完整文档，应用 `applied_style`
- 用户可以直接看到处理前后的对比
- 更直观地了解格式适配效果

**实现细节**：
```typescript
// 应用段落样式
const paragraphStyle: React.CSSProperties = {
  fontFamily: appliedStyle?.font?.name || '宋体',
  fontSize: `${appliedStyle?.font?.size || 12}pt`,
  fontWeight: appliedStyle?.font?.bold ? 'bold' : 'normal',
  textAlign: appliedStyle?.format?.alignment,
  lineHeight: appliedStyle?.format?.line_spacing || 1.5,
  marginBottom: `${appliedStyle?.format?.space_after || 0}pt`,
  textIndent: `${appliedStyle?.format?.first_line_indent || 0}pt`,
};
```

### 2. **两侧实时同步滚动**

**功能**：
- 用户滚动左侧文档，右侧自动跟随
- 用户滚动右侧文档，左侧自动跟随
- 使用防抖机制避免循环滚动

**实现**：
```typescript
useEffect(() => {
  const leftPanel = leftPanelRef.current;
  const rightPanel = rightPanelRef.current;

  if (!leftPanel || !rightPanel) return;

  let isScrolling = false;

  const handleLeftScroll = () => {
    if (isScrolling) return;
    isScrolling = true;

    const scrollRatio = leftPanel.scrollTop / (leftPanel.scrollHeight - leftPanel.clientHeight);
    rightPanel.scrollTop = scrollRatio * (rightPanel.scrollHeight - rightPanel.clientHeight);

    setTimeout(() => { isScrolling = false; }, 100);
  };

  const handleRightScroll = () => {
    if (isScrolling) return;
    isScrolling = true;

    const scrollRatio = rightPanel.scrollTop / (rightPanel.scrollHeight - rightPanel.clientHeight);
    leftPanel.scrollTop = scrollRatio * (leftPanel.scrollHeight - leftPanel.clientHeight);

    setTimeout(() => { isScrolling = false; }, 100);
  };

  leftPanel.addEventListener('scroll', handleLeftScroll, { passive: true });
  rightPanel.addEventListener('scroll', handleRightScroll, { passive: true });

  return () => {
    leftPanel.removeEventListener('scroll', handleLeftScroll);
    rightPanel.removeEventListener('scroll', handleRightScroll);
  };
}, []);
```

### 3. **编辑模式控制**

**设计理念**：
- **默认只读模式**：避免误操作
- **点击按钮进入编辑模式**：明确的用户意图
- **退出时确认未保存修改**：防止数据丢失

**UI布局**：

## 🔄 用户操作流程

### 场景1：浏览处理结果（只读模式）

1. 用户进入文档预览页面
2. 左侧显示原文档（原始样式）
3. 右侧显示处理后文档（新样式）
4. 用户可以滚动查看对比
5. 悬停段落查看样式信息

### 场景2：不满意处理结果（编辑模式）

1. 用户点击顶部"😕 对结果不满意？"按钮
2. 进入编辑模式：
   - 显示反馈面板
   - 显示操作按钮
   - 右侧文档可点击编辑
3. 用户填写反馈（可选）
4. 点击右侧段落
5. 弹出类型选择器
6. 选择新类型
7. 段落显示编辑标记（✏️）
8. 点击"💾 保存并重新生成"或"🤖 AI识别"
9. 文档重新生成，退出编辑模式

### 场景3：快速修正个别段落

1. 进入编辑模式
2. 点击需要修改的段落
3. 选择正确的类型
4. 点击"💾 保存"
5. 文档立即重新生成
6. 查看新效果

### 场景4：使用AI智能优化

1. 进入编辑模式
2. 填写反馈（例如："段落3应该是标题"）
3. 点击"🤖 AI识别"
4. 查看预览对话框
5. 确认后自动应用
6. 文档重新生成

## 🎨 设计原则

### 1. **直观性优先**
- 用户直接看到处理前后的对比
- 无需额外操作就能理解效果

### 2. **明确用户意图**
- 默认只读，避免误操作
- 明确的编辑模式切换
- 退出时确认未保存修改

### 3. **渐进式交互**
- 浏览（只读）→ 不满意（进入编辑）→ 修改（操作）→ 保存（应用）
- 每个步骤都有清晰的视觉反馈

### 4. **信息密度平衡**
- 移除不必要的搜索功能
- 精简结构信息显示
- 使用tooltip提供额外信息

### 5. **性能优化**
- 使用防抖避免频繁滚动事件
- passive事件监听器提升性能
