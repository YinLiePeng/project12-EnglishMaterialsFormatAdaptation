## 📋 用户需求概述

### 核心问题
用户对文档结构识别结果（无论是规则引擎处理的结果还是AI处理的结果）不满意时，需要一个便捷的方式来修正识别错误。

### 用户的具体想法

**原始需求描述：**
> "对规则引擎处理的结果，增加'结果不满意？尝试AI识别'的按钮，并为用户提供'意见反馈'的输入框（允许没有意见）。AI在原系统prompts加上用户意见反馈的指导下重新进行结构识别。对已经是AI处理的结果，增加'结果不满意？尝试重新识别'的按钮，并为用户提供'意见反馈'的输入框。AI在原系统prompts加上用户意见反馈的指导下重新进行结构识别。"

### 用户的关键决策

经过深入分析和讨论，用户明确了以下三个核心设计决策：

| 决策点 | 用户选择 | 说明 |
|--------|---------|------|
| **UI方案** | **A. 折叠面板** | 点击"结果不满意"展开修正选项，节省空间 |
| **智能判断** | **B. 用户选择** | 提供"快速修正"和"AI识别"两个按钮让用户选 |
| **预览机制** | **A. 总是预览** | 显示将要修改的段落，用户确认后才应用 |

---

## 🎯 功能需求总结

### 核心功能

1. **快速修正模式**
   - 用户可以直接点击段落的类型标签
   - 下拉选择新的类型
   - 立即生效（零延迟）
   - 支持批量修正

2. **AI识别模式**
   - 用户可以填写意见反馈（可选）
   - AI基于用户意见重新识别
   - 显示预览：列出所有将要修改的段落
   - 用户确认后才应用

3. **文档重新生成**
   - 修正后自动重新生成文档
   - 保持原始文件内容
   - 仅应用新的样式映射

### 非功能需求

- **性能**: 快速修正<1秒，AI识别2-10秒
- **可用性**: 操作简单，反馈及时
- **可靠性**: 总是预览，避免意外修改
- **成本控制**: API成本极低（¥0.004/文档）

## 🔧 后端API详细设计

### API端点1：快速修正

**端点**: `POST /api/v1/tasks/{task_id}/quick-correction`

**请求体**:
```json
{
  "paragraph_updates": [
    {
      "index": 0,
      "content_type": "title"
    },
    {
      "index": 5,
      "content_type": "heading"
    }
  ],
  "user_feedback": "段落3应该是标题"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "修正成功",
  "data": {
    "updated_count": 2,
    "structure_analysis": {
      "method": "rule_engine",
      "overall_confidence": 0.92,
      "paragraphs": [...]
    }
  }
}
```

**实现逻辑**:
```python
@router.post("/{task_id}/quick-correction")
async def quick_correction(
    task_id: str,
    request_data: dict,
    db: AsyncSession = Depends(get_db)
):
    # 1. 获取任务和structure_analysis
    # 2. 解析paragraph_updates
    # 3. 更新指定段落的content_type
    # 4. 设置confidence=1.0，reason="用户手动修正"
    # 5. 重新计算overall_confidence
    # 6. 保存到数据库
    # 7. 返回更新后的structure_analysis
```

## 📊 完整数据流

### 快速修正流程

```
用户操作：
1. 点击段落0的类型标签"题号"
2. 下拉选择"主标题"
3. 立即生效（前端乐观更新）
   → confidence变为1.0
   → 显示编辑标记✏️
   → 顶部显示"已编辑1处"

4. 点击"保存并重新生成"按钮
   ↓
前端API调用：
POST /api/v1/tasks/{id}/quick-correction
{
  "paragraph_updates": [
    {"index": 0, "content_type": "title"}
  ],
  "regenerate": true
}
   ↓
后端处理：
1. 获取任务的structure_analysis
2. 更新paragraphs[0]的content_type="title"
3. 设置confidence=1.0，reason="用户手动修正"
4. 保存到数据库
5. 重新生成文档（使用新的style_keys）
   ↓
返回结果：
{
  "updated_count": 1,
  "structure_analysis": {...},
  "output_file_url": "..."
}
   ↓
前端更新：
1. 刷新structure_analysis
2. 清空编辑状态
3. 下载按钮指向新文档
```

### AI识别流程

```
用户操作：
1. 展开"结果不满意？"面板
2. 填写意见："段落3应该是标题"
3. 点击"AI识别"按钮
   ↓
前端API调用：
POST /api/v1/tasks/{id}/ai-recognize
{
  "user_feedback": "段落3应该是标题",
  "mode": "preview"
}
   ↓
后端处理：
1. 获取任务和原始文档
2. 重新解析文档内容
3. 构建增强Prompt：
   """
   原始Prompt + 用户意见
   
   用户意见：段落3应该是标题
   
   请基于用户意见重新识别...
   """
4. 调用hybrid_recognizer(use_llm=True)
5. 对比新旧structure_analysis
6. 提取变化列表
   ↓
返回预览：
{
  "mode": "preview",
  "changes": [
    {index: 3, old_type: "body", new_type: "heading", ...},
    {index: 5, old_type: "body", new_type: "question_number", ...},
    ...
  ]
}
   ↓
前端显示预览对话框：
1. 列出所有将要修改的段落
2. 显示修改前后对比
3. 显示修改理由
4. 显示警告提示
   ↓
用户确认：
点击"确认并重新生成文档"按钮
   ↓
前端API调用：
POST /api/v1/tasks/{id}/ai-recognize
{
  "user_feedback": "段落3应该是标题",
  "mode": "apply"
}
   ↓
后端处理：
1. 重新执行AI识别（或复用之前的识别结果）
2. 保存structure_analysis到数据库
3. 自动调用regenerate_document()
   ↓
返回应用结果：
{
  "mode": "apply",
  "applied_count": 50,
  "structure_analysis": {...}
}
   ↓
前端更新：
1. 完全刷新structure_analysis
2. 清空所有编辑状态
3. 下载按钮指向新文档
```

### 前端：DocumentPreview.tsx 关键部分

```typescript
import { useState, useCallback } from 'react';
import { useParams } from 'react-router-dom';
import { quickCorrection, aiRecognize, regenerateDocument } from '../services/api';
import type { StructureAnalysis, ParagraphEdit } from '../types';

const CONTENT_TYPE_OPTIONS = [
  { value: 'title', label: '主标题' },
  { value: 'heading', label: '子标题' },
  { value: 'question_number', label: '题号' },
  { value: 'option', label: '选项' },
  { value: 'body', label: '正文' },
  { value: 'answer', label: '答案' },
  { value: 'analysis', label: '解析' },
];

export function DocumentPreview() {
  const { taskId } = useParams<{ taskId: string }>();
  const [analysis, setAnalysis] = useState<StructureAnalysis | null>(null);
  const [edits, setEdits] = useState<ParagraphEdit[]>([]);
  const [feedback, setFeedback] = useState('');
  const [isRecognizing, setIsRecognizing] = useState(false);
  
  // 快速修正：更新段落类型
  const handleQuickCorrection = async () => {
    if (edits.length === 0) return;
    
    try {
      const response = await quickCorrection(taskId, {
        paragraph_updates: edits.map(e => ({
          index: e.paragraphIndex,
          content_type: e.newType
        })),
        user_feedback: feedback
      });
      
      if (response.code === 0) {
        setAnalysis(response.data.structure_analysis);
        setEdits([]);
        
        // 自动重新生成文档
        await handleRegenerate();
      }
    } catch (error) {
      console.error('快速修正失败:', error);
      alert('修正失败，请重试');
    }
  };
  
  // AI识别
  const handleAIRecognize = async () => {
    setIsRecognizing(true);
    
    try {
      // 第一步：预览模式

        user_feedback: feedback,
        mode: 'preview'

    } finally {
      setIsRecognizing(false);

        mode: 'apply'

      if (applyResponse.code === 0) {
        setAnalysis(applyResponse.data.structure_analysis);
        setShowPreviewModal(false);
        setEdits([]);
        
        // 自动重新生成文档
        await handleRegenerate();
      }
    } catch (error) {
      console.error('应用修正失败:', error);
      alert('应用失败，请重试');
    }
  };

  const handleRegenerate = async () => {
```

```python
@router.post("/{task_id}/quick-correction")
async def quick_correction(
    task_id: str,
    request_data: Dict[str, Any],
    db: AsyncSession = Depends(get_db)
):
    """
    快速修正：直接更新用户指定的段落类型
    
    参数:
        paragraph_updates: 段落更新列表
        user_feedback: 用户意见（可选）
    
    返回:
        更新后的结构分析数据
    """
    # 获取任务
    result = await db.execute(select(Task).where(Task.task_id == task_id))
    task = result.scalar_one_or_none()
    
    if not task:
        raise HTTPException(status_code=404, detail="任务不存在")
    
    if not task.structure_analysis:
        raise HTTPException(status_code=400, detail="没有结构分析数据")
    
    # 解析当前结构分析
    structure = json.loads(task.structure_analysis)
    updates = request_data.get("paragraph_updates", [])
    
    if not updates:
        raise HTTPException(status_code=400, detail="没有提供更新内容")
    
    # 更新段落类型
    update_map = {u["index"]: u["content_type"] for u in updates}
    updated_count = 0
    
    for para in structure["paragraphs"]:
        if para["index"] in update_map:
            old_type = para["content_type"]
            new_type = update_map[para["index"]]
            
            # 更新类型
            para["content_type"] = new_type
            para["content_type_name"] = CONTENT_TYPE_NAMES.get(new_type, new_type)
            para["confidence"] = 1.0
            para["reason"] = f"用户手动修正（从{old_type}改为{new_type}）"
            
            # 更新applied_style
            style_key = document_processor._content_type_to_style_key(new_type)
            # ... 获取样式定义并更新 ...
            
            updated_count += 1
    
    # 重新计算整体置信度
    structure["overall_confidence"] = sum(
        p["confidence"] for p in structure["paragraphs"]
    ) / len(structure["paragraphs"])
    
    # 保存到数据库
    task.structure_analysis = json.dumps(structure, ensure_ascii=False)
    await db.commit()
    
    return {
        "code": 0,
        "message": f"修正成功，已更新{updated_count}个段落",
        "data": {
            "updated_count": updated_count,
            "structure_analysis": structure
        }
    }


@router.post("/{task_id}/ai-recognize")
async def ai_recognize(
    task_id: str,
    request_data: Dict[str, Any],
    db: AsyncSession = Depends(get_db)
):
    """
    AI重新识别：基于用户意见重新运行结构识别
    
    参数:
        user_feedback: 用户意见（可选）
        mode: "preview"（预览）或 "apply"（应用）
    
    返回:
        preview模式: changes列表 + 新的structure_analysis
        apply模式: 应用结果 + 新的structure_analysis
    """
    # 获取任务
    result = await db.execute(select(Task).where(Task.task_id == task_id))
    task = result.scalar_one_or_none()
    
    if not task:
        raise HTTPException(status_code=404, detail="任务不存在")
    
    if task.enable_llm != 1:
        raise HTTPException(status_code=400, detail="该任务未启用AI识别功能")
    
    user_feedback = request_data.get("user_feedback", "")
    mode = request_data.get("mode", "preview")
    
    # 重新解析文档
    parser = DocxParser(task.input_file_path)
    elements = parser.extract_content()
    para_dicts = document_processor._extract_para_dicts(elements)
    
    # 构建增强的Prompt
    enhanced_prompt = build_recognition_prompt(
        user_feedback=user_feedback,
        current_structure=task.structure_analysis
    )
    
    # 调用混合识别器（强制使用LLM）
    structures = await hybrid_recognizer.recognize(
        para_dicts,
        use_llm=True,
        custom_prompt=enhanced_prompt
    )
    
    # 格式化结果
    style_mapping = document_processor.get_style_mapping_for_task(task)
    paragraphs = extract_from_elements(elements)
    
    formatted = structure_formatter.format_rule_engine_results(
        structures, style_mapping, paragraphs
    )
    formatted["method"] = "llm"
    
    if mode == "preview":
        # 预览模式：对比变化
        current_structure = json.loads(task.structure_analysis)
        changes = compare_structures(current_structure, formatted)
        
        return {
            "code": 0,
            "message": f"识别完成，{len(changes)}处变化，请确认",
            "data": {
                "mode": "preview",
                "changes": changes,
                "structure_analysis": formatted  # 预览结果，未保存
            }
        }
    else:
        # 应用模式：直接保存
        task.structure_analysis = json.dumps(formatted, ensure_ascii=False)
        await db.commit()
        
        return {
            "code": 0,
            "message": "已应用AI识别结果",
            "data": {
                "mode": "apply",
                "applied_count": len(formatted["paragraphs"]),
                "structure_analysis": formatted
            }
        }

    user_feedback: str,

    base_prompt = get_prompt("structure_recognition")

        return enhanced

def compare_structures(
    old_structure: dict,
    new_structure: dict
) -> List[Dict[str, Any]]:
    """对比两个结构分析，返回变化列表"""
    changes = []
    
    old_paras = {p["index"]: p for p in old_structure["paragraphs"]}
    new_paras = {p["index"]: p for p in new_structure["paragraphs"]}
    
    for index in new_paras:
        new_para = new_paras[index]
        
        if index in old_paras:
            old_para = old_paras[index]
            
            # 检查是否有变化
            if old_para["content_type"] != new_para["content_type"]:
                changes.append({
                    "index": index,
                    "old_type": old_para["content_type"],
                    "old_type_name": old_para["content_type_name"],
                    "new_type": new_para["content_type"],
                    "new_type_name": new_para["content_type_name"],
                    "reason": new_para.get("reason", "AI重新识别")
                })
    
    return changes


async def regenerate_with_structure(
    task: Task,
    structure_analysis: dict,
    db: AsyncSession
) -> str:
    """使用更新后的结构重新生成文档"""
    # 解析原始文档
    parser = DocxParser(task.input_file_path)
    elements = parser.extract_content()
    
    # 构建新的style_keys
    style_keys = {}
    for para_info in structure_analysis["paragraphs"]:
        style_key = document_processor._content_type_to_style_key(para_info["content_type"])
        style_keys[para_info["index"]] = style_key
    
    # 获取样式映射
    preset_style = get_preset_style(task.preset_style or "universal")
    style_mapping = get_style_mapping(preset_style)
    
    # 生成新文档
    generator = DocxGenerator()
    generator.generate_from_elements(elements, style_mapping, style_keys)
    
    # 生成输出文件名
    output_path = document_processor._get_output_path(
        task.input_file_path,
        task.input_filename
    )
    
    generator.save(output_path)
    
    # 更新任务
    task.output_file_path = output_path
    task.output_filename = os.path.basename(output_path)
    await db.commit()
    
    return output_path
```

## ⚠️ 风险和缓解措施

### 风险1：乐观更新冲突

**风险描述**: 用户快速连续编辑多个段落，导致状态不一致

**影响**: 中等

**缓解措施**:
1. 使用防抖处理（300ms延迟）
2. 每次更新基于最新的analysis状态
3. 添加版本号或时间戳

### 风险2：LLM API限流

**风险描述**: AI识别触发API限流，返回错误

**影响**: 中等

**缓解措施**:
1. 添加重试机制（指数退避）
2. 显示友好的错误提示
3. 提供降级方案（建议手动修正）

### 风险3：文档重新生成失败

**风险描述**: 更新后重新生成文档时出错

**影响**: 中等

**缓解措施**:
1. 只更新structure_analysis（不重新生成）
2. 异步后台生成
3. 失败回滚机制

### 风险4：用户等待焦虑

**风险描述**: AI识别需要等待2-10秒，用户可能焦虑

**影响**: 高

**缓解措施**:
1. 实时显示识别进度
2. 预估剩余时间
3. 提供取消按钮
