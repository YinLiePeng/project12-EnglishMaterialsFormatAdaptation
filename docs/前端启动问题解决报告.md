## 🔍 问题分析

### 原始错误
```
Error: listen EACCES: permission denied ::1:5173
```

### 根本原因
不是端口占用问题，而是 **5173端口范围的权限问题**。

可能的原因：
1. **WSL网络配置限制** - WSL环境下某些端口范围可能被限制
2. **防火墙或安全软件** - 可能阻止了5173端口
3. **之前的进程残留** - 虽然没有检测到，但可能存在

---

## ✅ 解决方案

### 使用的解决方案：更换端口号

```bash
cd frontend
npm run dev -- --port 3000
```

### 方案2：修复5173端口权限问题（推荐）

#### 选项A：检查WSL网络配置
```bash
# 检查WSL版本
wsl --version

# 检查网络配置
cat /etc/wsl.conf
```

#### 选项B：检查防火墙设置
```bash
# Windows PowerShell（管理员权限）
netsh interface ipv4 show excludedportrange protocol=tcp
```

#### 选项C：修改Vite配置
```bash
# 编辑 frontend/vite.config.ts
export default defineConfig({
  server: {
    port: 3000,  # 使用3000作为默认端口
    strictPort: false,  # 如果端口被占用，自动尝试下一个
    host: true  # 允许外部访问
  }
})
```
