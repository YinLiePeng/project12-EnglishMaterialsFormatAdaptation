# 文档预览界面UX改进 - 测试指南

## 🧪 测试环境准备

### 1. 启动服务

```bash
# 终端1：启动后端
cd backend
source venv/bin/activate  # Windows: venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 终端2：启动前端
cd frontend
npm run dev
```

### 2. 访问应用

- 前端地址：http://localhost:5173
- 后端地址：http://localhost:8000

## ✅ 测试场景

### 场景1：浏览处理结果（只读模式）

**目的**：验证用户可以直观地看到处理前后的对比

**步骤**：
1. 访问 http://localhost:5173
2. 上传一个测试文档
3. 等待处理完成，点击"查看详情"
4. 观察界面：
   - ✅ 左侧标题显示"原文档（处理前）"
   - ✅ 右侧标题显示"处理后文档"
   - ✅ 右侧文档应用了新样式（字体、大小、对齐等）
   - ✅ 顶部导航栏有"😕 对结果不满意？"按钮（outline样式）
   - ✅ 没有"已编辑"提示
   - ✅ 没有保存/AI识别按钮

**测试点**：
- 左右两栏等宽
- 右侧文档样式正确应用
- 悬停段落显示样式信息tooltip
- 滚动一侧，另一侧同步滚动

**预期结果**：
```
✅ 界面整洁，只读模式
✅ 用户可以直观对比处理前后
✅ 悬停显示tooltip
✅ 同步滚动流畅
```

   - ✅ 右侧标题变为"处理后文档（编辑模式）"

   - ✅ 右侧文档可以点击

**反馈面板验证**：
- ✅ 显示说明文字
- ✅ 有输入框（可以填写反馈）
- ✅ 有操作提示（3条提示文字）

**预期结果**：
```
✅ 按钮状态正确切换
✅ 反馈面板正确显示
✅ 右侧文档可交互
```

### 场景3：编辑段落类型

**目的**：验证段落类型编辑功能

**步骤**：
1. 在编辑模式下，点击右侧任意段落
2. 观察弹出框：
   - ✅ 显示"选择段落类型："标题
   - ✅ 显示7个类型按钮（网格布局）
   - ✅ 当前类型高亮显示（蓝色边框）
   - ✅ 按钮有不同颜色（标题紫色、正文灰色等）
3. 点击不同的类型按钮
4. 观察变化：
   - ✅ 弹出框关闭
   - ✅ 段落显示琥珀色边框（ring-2 ring-amber-400）
   - ✅ 段落右上角显示✏️图标
   - ✅ 顶部显示"已编辑 1 处"
   - ✅ "💾 保存并重新生成 (1)"按钮出现

**测试点**：
- 连续修改多个段落
- 点击已编辑的段落
- 选择器显示"恢复原始类型"按钮
- 点击恢复按钮，编辑标记消失

**预期结果**：
```
✅ 类型选择器正确显示
✅ 编辑标记正确显示
✅ 已编辑计数正确
```

```
✅ 同步滚动流畅
✅ 无明显延迟
✅ 无循环滚动问题
```

**目的**：验证快速修正功能

   - ✅ 等待处理完成
   - ✅ 显示"文档已重新生成"提示
   - ✅ 退出编辑模式
   - ✅ 右侧文档更新为新样式
   - ✅ 编辑标记消失

**验证点**：
- 修改的段落样式正确应用
- 未修改的段落保持不变
- 可以下载新文档

**预期结果**：
```
✅ 保存成功
✅ 文档重新生成
✅ 样式正确应用
```

### 场景7：AI识别

**目的**：验证AI智能识别功能

**步骤**：
1. 在编辑模式下，填写反馈（可选）
2. 点击"🤖 AI识别"按钮
3. 观察变化：
   - ✅ 按钮变为"🤖 识别中..."
   - ✅ 等待2-10秒
   - ✅ 显示预览对话框
   - ✅ 列出所有变化
   - ✅ 点击"确认并重新生成文档"
   - ✅ 文档重新生成
   - ✅ 退出编辑模式

**预览对话框验证**：
- 显示变化数量
- 每个变化显示：
  - 段落编号
  - 原类型 → 新类型
  - 变化理由
- 有警告提示
- 可以取消

**预期结果**：
```
✅ AI识别正常工作
✅ 预览对话框正确显示
✅ 应用后文档更新
```

### 场景8：退出编辑模式

**目的**：验证退出编辑模式的逻辑

**步骤1：无未保存修改**
1. 在编辑模式下，不进行任何修改
2. 点击"✓ 完成编辑"按钮
3. 观察：
   - ✅ 直接退出编辑模式
   - ✅ 按钮变回"😕 对结果不满意？"
   - ✅ 反馈面板消失
   - ✅ 右侧回到只读模式

**步骤2：有未保存修改**
1. 在编辑模式下，修改几个段落
2. 点击"✓ 完成编辑"按钮
3. 观察：
   - ✅ 显示确认对话框
   - ✅ 提示"您有 N 处未保存的修改，确定要退出编辑模式吗？"
   - ✅ 点击"确定"，退出并清空编辑
   - ✅ 点击"取消"，保持编辑模式

**预期结果**：
```
✅ 无修改时直接退出
✅ 有修改时显示确认
✅ 编辑状态正确管理
```

## 🐛 已知问题和限制

### 当前限制
1. **大文档性能**：100+段落的文档可能有性能问题
2. **移动端适配**：界面主要针对桌面端优化
3. **撤销功能**：目前不支持撤销操作

### 已解决的问题
1. ✅ 循环滚动问题（使用防抖）
2. ✅ 编辑模式状态管理（明确切换）
3. ✅ 样式应用问题（正确映射所有样式属性）
